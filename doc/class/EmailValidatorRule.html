# (void) constructor(options)

Create a new EmailValidatorRule. As a default it uses the W3C recommendation ^[a-zA-Z0-9.!#$%&'+/=?^_`{|}~-]+@a-zA-Z0-9?(?:.a-zA-Z0-9?)$ Please note that this requirement is a willful violation of RFC 5322, both accepting some addresses that would be invalid under RFC 5322, as well as rejecting some addresses that are valid within RFC 5322.
